== Introduction

This guide is for developers looking to integrate advanced search capabilities, text analytics, and topic-based content delivery into existing products, or build new solutions with InfoNgen.

This guide contains:

* General information about InfoNgen and a high-level overview of its APIs.
* An overview of the search functionality.
* Information about InfoNgen API authentication.
* A list of possible InfoNgen API errors, and the basic information on how to troubleshoot them.
* Eight API usage scenarios.
* Lists of InfoNgen APIs and their endpoints.
* A glossary of terms.

== Document conventions

* Double braces `{\{text}}` enclose a placeholder value that needs to be replaced with user-specific data in the request. +
For example, `{\{access_token}}` is a placeholder for an authorization token.
* All API request and response examples in the manual are https://curl.haxx.se[cURL] statements.
* _Italicized text^?^_ with a superscript question mark next to it marks the first mention of an item from the <<#glossary,Glossary>>. +
All subsequent instances of this Glossary item are then marked with _italicized text_ throughout this Guide.
* *Bold* text introduces *Notes*, *Figures*, and *Tables*, and denotes *points of emphasis*, where appropriate.
* Ellipses "..." mark truncation in code samples, figures, and text.
`"{...}"` mark truncated JSON objects, `"[...]"` mark truncated JSON arrays.
* Clickable expands with the text "Click here for ..." contain code samples, example requests, example responses, or additional information. +
